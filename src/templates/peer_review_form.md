## Оценочный лист (10-балльная шкала)

Оцените каждый критерий от 0 до 10. **Комментарий обязателен для баллов ≤ 5 и ≥ 9.**

| № | Критерий | Балл (0–10) | Комментарий (обоснование) |
|---|----------|-------------|---------------------------|
| **P1** | **Ясность постановки задачи** – чётко ли сформулирована цель, целевой пользователь, метрики успеха? | | |
| **P2** | **Обоснованность выбора методов Монте-Карло** – почему выбран именно этот метод (MCMC, HMC, MCTS, Particle Filter и др.)? Есть ли альтернативы? | | |
| **P3** | **Работа с данными** – корректно ли сгенерированы/предобработаны данные? Есть ли проверка качества? | | |
| **P4** | **Архитектура и качество кода** – структура кода, читаемость, комментарии, обработка ошибок, использование библиотек (NumPy, PyMC, PyTorch). | | |
| **P5** | **Воспроизводимость** – работает ли проект по инструкции? Есть ли файлы зависимостей (requirements.txt)? | | |
| **P6** | **Эксперименты и метрики** – релевантны ли метрики (сходимость MCMC, R-hat, ESS, точность/полнота)? Проведено ли сравнение с бейзлайном? | | |
| **P7** | **Репозиторная документация** – наличие и качество README, PROJECT_BRIEF, architecture.md, инструкций. | | |
| **P8** | **Анализ ограничений и рисков** – описаны ли границы применимости, возможные проблемы (вырождение частиц, плохая сходимость, выбор proposal)? | | |
| **P9** | **Соответствие выбранной ИИ-роли КРМ** – насколько проект демонстрирует индикаторы заявленной роли (см. ролевой блок ниже). | | |
| **P10** | **Общая инженерная культура** – использование Git, структура репозитория, наличие тестов (опционально), лицензия. | | |
