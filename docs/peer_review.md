# Регламент peer-review курсовых проектов

## 1. Общие положения
Peer-review (взаимное рецензирование) — обязательное формирующее оценочное средство (ОС-3), направленное на развитие у студентов навыков критического анализа, аргументации и профессиональной коммуникации. Рецензирование проводится **анонимно** (рецензент не знает автора, автор не знает рецензента) или **открыто** — по решению преподавателя.

## 2. Назначение рецензий
- Каждая проектная работа получает **2 независимые рецензии**.
- Каждый студент рецензирует **2 работы** (назначается преподавателем).
- Распределение осуществляется случайным образом или по принципу «каждый рецензирует работы двух других студентов» (перекрёстное рецензирование).
- Рецензент не должен иметь конфликта интересов с автором (не быть в одной команде, не быть соавтором).

## 3. Порядок проведения
1. Преподаватель формирует пары/тройки и публикует список назначений в ЭИОС или в репозитории курса.
2. Рецензент получает доступ к репозиторию автора и инструкции по запуску проекта.
3. Рецензент заполняет форму peer-review (шаблон `src/templates/peer_review_form.md`) в течение **5 календарных дней** после назначения.
4. Заполненная форма сохраняется в репозитории автора (в папке `reviews/`) или передаётся преподавателю.
5. Автор получает обезличенные рецензии (если рецензия анонимная) и имеет право на краткий ответ (rebuttal) в течение 2 дней.
6. Преподаватель оценивает качество каждой рецензии по 10-балльной шкале (см. раздел 6).

## 4. Требования к рецензенту
- Рецензия должна быть **конструктивной** и **аргументированной**.
- Каждый критерий оценивается баллом от 0 до 10 с обязательным комментарием при баллах ≤ 5 и ≥ 9.
- Рецензент обязан проверить:
  - работоспособность кода по инструкции;
  - наличие всех обязательных артефактов (README, PROJECT_BRIEF, architecture.md);
  - корректность экспериментов и метрик;
  - соответствие выбранной ИИ-роли КРМ.
 
**Критерии оценивания (10-балльная шкала):**

| Критерий | Что оценивается |
|----------|-----------------|
| P1: Постановка задачи | Чёткость, целевая аудитория, метрики успеха |
| P2: Обоснование метода | Почему выбран данный метод (генератор, MCMC, QMC, EM, имитация отжига, MCTS и т.д.)? Рассмотрены альтернативы? |
| P3: Работа с данными | Качество генерации/предобработки, проверка качества |
| P4: Архитектура и код | Структура, читаемость, комментарии, обработка ошибок |
| P5: Воспроизводимость | Работает ли по инструкции? Зависимости? |
| P6: Эксперименты | Метрики (сходимость, R-hat, ESS, силуэт), сравнение с бейзлайном |
| P7: Документация | README, PROJECT_BRIEF, архитектурный обзор |
| P8: Анализ ограничений | Границы применимости, улучшения |
| P9: Соответствие роли КРМ | Насколько проект подтверждает индикаторы роли |
| P10: Инженерная культура | Git, структура, лицензия |

## 5. Сроки и санкции
- За несоблюдение сроков рецензирования балл за peer-review снижается на 2 балла.
- За формальное или некачественное рецензирование (отсутствие комментариев, необоснованные баллы) рецензия может быть отправлена на доработку, а балл снижен.
- Рецензия, не содержащая содержательных комментариев, оценивается в 0 баллов.

## 6. Оценивание качества рецензии преподавателем
Преподаватель оценивает каждую рецензию по 10-балльной шкале по следующим критериям:

| Критерий | Балл (0–10) |
|----------|-------------|
| **Полнота** – все разделы формы заполнены | |
| **Аргументированность** – баллы подкреплены конкретными примерами и ссылками на код/документацию | |
| **Конструктивность** – рекомендации конкретны и выполнимы | |
| **Соответствие ролевому блоку КРМ** – оценка уровня роли обоснована индикаторами | |
| **Глубина анализа** – выявлены не только явные проблемы, но и потенциальные риски/улучшения | |
| **Общая культура рецензии** – вежливый, профессиональный тон, отсутствие оценочных суждений без доказательств | |

**Итоговый балл за рецензию** = среднее арифметическое по критериям, округлённое до целого.  
Этот балл влияет на оценку студента как рецензента в рамках ОС-3.

## 7. Итоговый вклад peer-review в оценку по дисциплине
- Peer-review оценивается как отдельное оценочное средство (ОС-3) с весом **10%** от итоговой оценки по дисциплине.
- Оценка складывается из:
  - качества написанной рецензии (70%);
  - активности и своевременности (30%).
